# HTML Advanced

## 📖 Project Overview

This project focuses on recreating a webpage from a designer-provided Figma file using **pure semantic HTML**.

The goal is to build a clean, accessible, and well-structured webpage without using CSS or JavaScript. Emphasis is placed on understanding document structure, semantic tags, accessibility, and best practices in modern web development.

This project serves as the foundation for future stages where styling and interactivity will be added.

---

## 🎯 Learning Objectives

By completing this project, I aim to:

* Understand the importance of semantic HTML.
* Structure a webpage according to a professional design specification.
* Improve accessibility through meaningful HTML elements.
* Learn how to interpret and implement layouts from Figma designs.
* Build maintainable and scalable web page structures.

## 🏗 Features Implemented

### Semantic Structure

The webpage is built using semantic HTML elements such as:

* `<header>`
* `<nav>`
* `<main>`
* `<section>`
* `<article>`
* `<aside>`
* `<footer>`

### Accessibility Considerations

* Proper heading hierarchy.
* Meaningful alternative text for images.
* Logical content organization.
* Semantic navigation structure.

### Content Sections

The webpage includes:

* Header and navigation
* Hero section
* Feature sections
* Testimonials
* Media content
* Footer information
